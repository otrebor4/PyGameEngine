Game
Base class that handle main loop, initialize screen, resources, read keyboard state
VariablesDescription
RESOLUTIONScreen resolution
GAMENAMEName to displace on caption
resourcesobject type Resources()
screenmain display
keyboardobject type KeyBoard()
worldobject type World()

MethodsDescription
__init__(self)initialize game, set world,keyboard,resources and screen to it initial values
draw(self,deburg)draw method, is called to draw world to screen, if deburg is set to True it will draw collider box.
update(self,delta)main update call, read keyboard input and saved to self.keyboard call world.update(delta)
run(self)main executable method, it start game, load if is neded and handle game loop

KeyBoard
class that state and update keyboard reads.
MethodsDescription
__init__(self)reset keyState hash
update(self)read current keyboard state and shash it to keyState
isKeyDown(self,key)check if a key was presed down since the last update, return false if the key wasn't on the keyState, else return keyState value. key is a pygame key value

Terrain
VariablesDescription
size(x,y), terrain dimensions
layers[], terrain layers data, a layer is an image.
resourcesclass type Resources()
worldclass type World(), world where the terrain belong
MethodsDescription
__init__(self,res,size,world)initialize terrain, res is the Resources class to use; default new Resources(), size is the dimension of the terrain; default (512,512), world where the terrain belong, default None.
GenerateLayer(self,tyles)generate a image layer from a set of tyleInfo, return the Image created
AddLayer(self,map_file,haveCollision)generate a layer getting the tyleInfo from a text file, if haveCollision is True then it will geretate collider blocks for the tyles. map_file filename string to get data from, haveCollision determine if generate collisoin to layer
drawLayer(self,screen,args)draw specific layers to an image 'screen', screen  destination were to draw layers, args a list of integer for layer to draw. 

World
VariablesDescription
terrainTerrain data/layers
phyEngPhysic Engine related to the world
objectslist of game objects that are active on the world
toRemovelist of game objects that will be removed by the end of the update
gravityvector2(), of the force of gravity on the world
mainobject of class Game().
MethodsDescription
__init__(self,main,gravity)initialize world on Game() main with gravity, default gravity = Vector2(0,0)
LoadTerrain(self,map_file,haveCollision)add a layer to the terrain, using the data from map_file (string), if haveCollision is True it will tell the terrain to generate colliders for the tyles generated on the new layer
createPolygon(self, points, static)create a new gameObject with shape polygon and collider polygon, points a set of (x,y).
createWall(self,x,y,w,h,color)create a wall on the with values x,y,w,h,color. where x is x position, y is y positoin, w is width of the rectangle, h is the height of the wall. color for the wall
createCircle(self,x,y,r,color,riged)create a Circle on the position x,y with radius y. color of the circle, and if riged is True it will add a compoment Riged to the Circle.
createRect(self,x,y,w,h,color,riged)create a rectangle at positoin x,y with widht w and height h, with a color, if riged is True it will add a component Riged to the Rectangle
AddObject(self,gameObject)add gameObject to be part of the world, if the object have collider it will added to the physEng
Delete(self,gameObject)remove gameObject's collider from the physEng and mark the object to be removed by the end of the update frame
update(self,delta)update game objects on the world, then it call physEng to update, finish by removing gameobjects marked to remove
updateRemove(self)remove game objects, and clean remove list
debDraw(self,screen)make phyEng to draw colliders
draw(self,screen)draw first two layer from the terrain, then draw each gameObject, finaly draw last two layer from terain

GameObject
GameObject, base class for every object on the world.
VariablesDescription
worldobject type World(), is the world in wich the gameObject belong.
shapeobject type Shape(),
colliderobject type Collider()
rendera componet with draw function, by default is None.
componentscomponents, a set of component atached to the game object
rigedobject type Riged(), a component that handle pyshics responses.
mainobject type Game(), link to Game() class instance
MethodsDescription
__init__(self,world)initialize gameobject and set world to be world. world is a World().
addComponent(self,component)create a component of typeClass component and added to the gameObject, component is classType.
getComponent(self,name)return a component from componets with the class.__name name, if the component is not found return None
getComponents(self,name)return a list of component from componets with the class.__name name.
update(self,delta)update call for the gameObject, it call update(delta) on each component attached
draw(self,screen)draw call, if the object contain a render, it will call the method draw(screen) on the render component
sendMessage(self,fname,arg)For each component on the gameobject it will call the method given by fname, and pass the arg as arguments for the method


RectObject
Extended GameObject with existing component collider and render
VariablesDescription
colorcolor(r,p,g,a) to be used to draw rectangle
colliderset GameObject.collider to a RectCollider
renderset render component to an component that draw the rectangle
MethodDescription
__init__(self,world,x,y,w,h,color)initialize a RectObject, with the values x as x position, y as y position, w as width, h as the height and color to be draw.
rect(self)return a pygame.Rect() object with the dimension of the gameObject
drawRect(self, screen)method that draw rectObject to the screen
